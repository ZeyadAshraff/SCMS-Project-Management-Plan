##  About This Repository

This repository contains the full **Project Management Plan (PMP)** for the School Course Management System (SCMS) — a web-based academic platform designed to centralize course registration, gradebook management, and user authentication for a single school.

The focus of this project is the **management and planning process**, not the implementation. All documents here reflect applied SPM concepts including estimation models, scheduling, resource allocation, and risk analysis.

##  Plan Structure

### Section 1 — Overview
- **Purpose & Scope:** Centralizing course catalog, enrollment, clash detection, and gradebook for one school over one academic term.
- **Objectives:** Includes measurable targets such as ≥80% reduction in scheduling conflicts, ≥50% reduction in registrar workload, and ≤1s catalog query response time.
- **Assumptions & Constraints:** Covers time, budget, scope, technology stack, deployment, integration, performance, and data privacy constraints.
- **Project Deliverables:** Defined across four phases — Analysis, Design, Development, and Testing.

### Section 4 — Project Organization
- **Roles & Responsibilities:** Project Manager, Analysis Manager, Design Supervisor, Development Manager, and Testing Manager — each with clearly scoped responsibilities.

### Section 5 — Management Process
- **Estimation:** Function Point Analysis using a conversion factor of 46 LOC/FP (Java baseline), applied across all major system functions including course registration, clash detection, gradebook, authentication, and reporting.
- **Work Activities:** Full task breakdown per phase.
- **Scheduling:** Timeline and milestone planning across the 8-week project window.
- **Resource Allocation:** Team assignments mapped to phases and deliverables.
- **Risk Management:**
  - Risk identification across technical, organizational, and process categories.
  - Quantitative risk analysis using Probability × Loss exposure calculations.
  - Risk Reduction Leverage (RRL) computed for each mitigation strategy.
  - Contingency plans defined for all high-priority risks.

### Section 6 — Technical Process
- **Methodology:** PMBOK for project management, Agile Scrum for software development.
- **Tools:** Git/GitHub, Figma, Draw.io, VS Code, Postman, Jest, Jira/Trello, Google Docs.
- **Standards:** ESLint/Prettier, UML/ERD modeling, UAT, role-based access control, encrypted storage.
